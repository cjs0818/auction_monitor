# 법원 경매 시각자료 및 다음 입찰가격 수정

## 수정 원인

### 1. 법원 경매 사진

법원 조건검색·사건조회 JSON에는 온비드의 `thnlImgUrlAdr`와 같은 대표사진 URL이 없다. 이전 코드는 사건화면에서 사진 버튼을 찾다가 사건검색 폼이나 탭을 찾지 못하면 즉시 종료하여 카드가 계속 빈 이미지로 남을 수 있었다.

이번 수정본은 다음 순서로 대표 시각자료를 보강한다.

1. 법원 사건화면에 이미 표시된 이미지·캔버스 탐색
2. 대상 물건 행과 물건내역 탭 탐색
3. 물건사진·현황사진 버튼 탐색
4. 감정평가서·현황조사서·매각물건명세서 미리보기 탐색
5. 공식 시각자료가 없거나 법원 UI 탐색에 실패하면 네이버 소재지 위치지도 표시
6. 현황사진이 없으면 위치지도 표시

카드 오른쪽 위에는 출처가 `현황사진` 또는 `위치지도`로 구분된다. 감정평가서·현황조사서 등 법원문서는 대표 이미지로 사용하지 않는다.

### 2. 법원 경매 최저매각가격

법원 조건검색 응답의 `minmaePrice`가 직전 유찰회차 가격으로 남는 사례가 있어 다음 공고가격 필드인 `notifyMinmaePrice1~4`를 먼저 사용하도록 수정했다.

또한 최종 표시후보는 사건상세의 기일내역에서 다음 조건을 만족하는 행으로 다시 교정한다.

- 같은 물건번호
- 오늘 이후 또는 목록의 예정 매각기일과 일치
- 유찰·매각·취하 등 종료결과가 없는 행 우선
- 가장 가까운 예정 매각기일

그 행의 매각기일과 최저매각가격을 함께 사용하므로 직전 유찰가격과 다음 입찰가격이 섞이지 않는다.

### 3. 온비드 공매 최저입찰가

온비드는 동일 물건의 회차별 행 중 현재 진행 중이거나 가장 가까운 유효 입찰마감일을 먼저 고른 뒤, 반드시 그 행에서 가격을 읽는다.

가격 해석 순서는 다음과 같다.

1. 명시적 현재 최저입찰가 필드
2. 최저입찰가 안내문구의 실제 금액
3. 같은 행의 최초입찰가
4. 감정평가액 대비 최저입찰가 비율 계산

`80%` 같은 비율 문구를 `80원`으로 오인하지 않도록 방어 로직도 추가했다.

## 적용 후 해야 할 일

기존 사진·검색 응답 캐시에는 예전 가격과 사진 실패 결과가 남을 수 있으므로 패치 스크립트가 다음 캐시를 자동으로 비운다.

- `data/court_photo_cache/`
- `data/selenium_cache/`
- `data/onbid_cache_v2/`

SQLite 검색결과와 검색조건은 삭제하지 않는다. 패치 후 `검색 캐시를 무시하고 새로 조회`를 선택하여 경매·공매 검색을 한 번 다시 실행한다.

## 진단정보

검색 진단에는 다음 항목이 추가된다.

- 경매사진 캐시
- 경매사진 신규수집
- 경매사진 실패
- 경매가격 상세교정
- 경매가격 교정실패

상세페이지 원시정보에는 가격 출처가 저장된다.

- 법원: `court_list_price_source`, `court_price_source`, `court_selected_schedule`
- 온비드: `onbid_price_source`, `onbid_selected_round_end`, `ONBID_SCHEDULE_ROWS`
