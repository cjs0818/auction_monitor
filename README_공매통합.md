# 온비드 공매 통합 기능

## 화면 선택

대시보드 첫 화면의 `검색대상`에서 `경매`, `공매`, `경매 및 공매` 중 하나를 선택합니다. 선택값은 저장되며 자동 실행에도 사용됩니다.

## 공매 인증키

`수집·알림 설정 → 온비드 공매 설정`에서 공공데이터포털의 `캠코공매물건조회서비스` 일반 인증키를 입력합니다. API 키는 이 Mac의 `config/config.yaml`에 저장되지만 사용자가 파일을 직접 편집할 필요는 없습니다.

## 공통 조건 적용

기존 검색조건의 지역, 물건용도, 진행상태, 가격, 감정평가액, 할인율, 유찰횟수, 면적, 일정, 포함·제외 키워드가 공매에도 적용됩니다.

온비드 서버에는 지역·가격·입찰기간을 우선 전달하고, 반환된 결과를 LandWatch의 공통 필터와 투자점수 평가에 다시 적용합니다.

## 공매 번호 체계

- 사건/공고번호: 온비드 `공고번호`
- 물건번호/물건관리번호: 온비드 `물건관리번호`
- 원시정보에서 공매번호와 내부 물건번호를 추가 확인

## 상세조회

공고번호 링크를 누르면 새 탭에서 온비드 공매 상세정보를 표시합니다. 상세조회 버튼을 통해 기본정보와 입찰 회차별 일정도 불러옵니다.


## 인증키 오류 30 점검

프로그램이 호출하는 API는 `ThingInfoInquireSvc/getUnifyUsageCltr`이며, 공공데이터포털에서
**한국자산관리공사 물건정보조회서비스** 활용신청이 승인되어 있어야 합니다.
온비드코드조회서비스 또는 다른 공매 API만 승인된 경우 같은 포털 인증키라도 오류 30이 발생할 수 있습니다.
수정본은 Encoding/Decoding 키와 전체 URL 붙여넣기를 자동 정리하고 두 전송방식을 순차 시험합니다.
오류 메시지에는 비밀키 원문 대신 길이와 SHA-256 앞 10자리만 표시됩니다.
