# 온비드 원문 링크 재수정

## 원인

차세대 온비드 OpenAPI가 제공하는 `cltrMngNo`와 `pbctCdtnNo`는 OpenAPI 상세조회용 식별자입니다.
기존 프로그램은 이 값과 목록의 다른 식별자를 조합하여 구 온비드 웹 상세주소
`collateralRealEstateDetail.do`를 만들었으나, 웹 상세화면이 요구하는 내부 식별자와 의미가 달라
`요청하신 페이지를 찾을 수 없습니다`가 발생했습니다.

대표 재현 물건: `2025-12641-001`

## 수정

- 차세대 API 값으로 구 웹 상세주소를 추정하지 않습니다.
- 온비드가 응답에 명시적인 공식 웹 URL을 제공한 경우에만 그 URL을 사용합니다.
- 그 외에는 현재 온비드 공식 부동산 물건목록 화면을 엽니다.
- 상세화면에 물건관리번호를 복사할 수 있도록 코드 블록으로 표시합니다.
- 기존 DB 또는 세션에 구 상세주소가 남아 있어도 화면 단계에서 공식 물건검색 주소로 대체합니다.

공식 물건검색 주소:

`https://www.onbid.co.kr/op/cta/cltrdtl/collateralDetailRealEstateList.do`

## 사용 방법

1. 공매 물건 상세화면에서 `온비드 부동산 물건검색 열기`를 누릅니다.
2. 함께 표시되는 물건관리번호를 복사합니다.
3. 온비드 물건검색 화면의 물건관리번호 조건에 붙여넣어 조회합니다.

## 검증

- `2025-12641-001` 전용 회귀 테스트 추가
- 구 `collateralRealEstateDetail.do` 주소가 생성되지 않는지 확인
- 기존 잘못된 DB URL이 화면에서 재사용되지 않는지 확인
- 전체 자동 테스트 119개 통과
