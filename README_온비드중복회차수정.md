# 온비드 공매 동일 물건 중복 표시 수정

## 원인

차세대 온비드 부동산 물건목록 API는 동일한 `물건관리번호(cltrMngNo)`에 대해 현재 회차와 향후 예정 회차를 각각 별도 행으로 반환할 수 있습니다.

기존 프로그램은 공매물건 식별키에 `공매조건번호(pbctCdtnNo)`를 포함했기 때문에, 하나의 토지가 회차별로 서로 다른 물건처럼 저장·표시되었습니다.

예를 들어 `2025-12641-001`은 동일한 토지이지만 다음 입찰회차가 별도 행으로 들어왔습니다.

- 2026-06-17 / 10,511,000원
- 2026-06-24 / 8,759,000원
- 2026-07-01 / 7,007,000원
- 2026-07-08 / 5,256,000원

## 수정 내용

- 공매 고유 식별키를 `물건관리번호` 기준으로 변경
- 같은 물건관리번호의 여러 회차를 한 건으로 병합
- 대표 회차는 다음 순서로 선택
  1. 오늘 이후의 진행 가능한 회차
  2. 가장 가까운 입찰마감일
  3. 동일 날짜이면 진행 중인 회차 우선
- 향후 예정 회차는 삭제하지 않고 상세페이지의 `입찰회차 목록`에 보존
- 과거 버전에서 SQLite DB에 저장된 회차별 중복행 자동 정리
- 진단정보에 `중복회차병합` 건수 표시

## 적용 후

결과표에는 `2025-12641-001`이 한 번만 표시됩니다. 현재 날짜가 2026-06-16이면 대표값은 다음과 같습니다.

- 입찰마감일: 2026-06-17
- 최저입찰가: 10,511,000원
- 진행상태: 진행

상세페이지 원시정보에서는 총 4개 입찰회차를 모두 확인할 수 있습니다.

## 적용 후 조치

프로그램을 다시 시작하고 공매 검색을 한 번 실행하십시오. 기존 DB의 회차별 중복행은 자동으로 정리됩니다. 온비드 캐시는 삭제할 필요가 없습니다.
