# 온비드 공매 시·도 전체검색 수정

## 문제

차세대 온비드 부동산 목록 API에 `lctnSdnm=경기도`처럼 시·도만 전달하면,
`lctnSggnm=가평군`을 함께 전달한 시·군·구 검색에서는 조회되는 물건이 있음에도
시·도 단독 검색은 0건으로 반환되는 경우가 있었다.

## 수정 동작

1. 시·도 단독 조건으로 먼저 온비드 API를 조회한다.
2. 시·도 직접조회가 0건이면 전국 목록을 1,000건/페이지로 조회한다.
3. 전국 응답 중 정규화된 소재지가 선택한 시·도와 일치하는 물건만 보존한다.
4. 전국 전체건수가 설정된 페이지 상한을 넘으면 해당 시·도의 시·군·구를 분할 조회하여 누락을 보완한다.
5. 시·군·구를 직접 선택한 검색은 기존처럼 해당 지역만 조회하며 전국 대체검색을 수행하지 않는다.

## 검색 진단

대체 경로가 사용되면 검색 진단에 다음 중 하나가 표시된다.

- `차세대 온비드 전국 대체검색 후 시·도 주소검증`
- `차세대 온비드 시·군·구 분할 대체검색`

지역 필터는 물건의 시·도, 시·군·구 및 소재지 주소를 정규화하여 검증하므로
다른 시·도의 물건은 결과에서 제외된다.
