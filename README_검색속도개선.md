# 검색속도 개선

## 기존 검색이 약 1분 30초 걸린 이유

90일 조회는 법원 검색 안정성을 위해 14일 단위 7개 구간으로 나눕니다.
전북특별자치도와 강원특별자치도는 기존 버전에서 현행 코드와 과거 코드를 항상 모두 조회했습니다.
따라서 전북 부안군 90일 검색은 첫 페이지만 조회해도 최소 14회 요청이 발생했습니다.

기본 대기시간은 요청 사이 3~4.5초이므로 대기시간만 약 49초입니다. 여기에 Chrome 시작,
검색화면 준비 6초, 각 법원 응답시간이 더해져 80~100초가 소요될 수 있습니다.

## 개선 내용

1. 현행 지역코드에서 대상지역 물건이 확인되면 과거 코드는 조회하지 않습니다.
   - 정상적인 전북 부안군 90일 검색: 14회 → 7회
   - 현행 코드가 0건일 때만 과거 코드로 다시 조회
2. 고정 6초 준비 대기를 제거하고 페이지가 준비되면 약 0.75초 후 검색을 시작합니다.
3. 동일한 요청 응답을 기본 15분간 저장합니다.
   - 동일 조건 반복검색은 법원 재요청 없이 캐시를 사용
   - 첫 화면에서 `검색 캐시를 무시하고 새로 조회`를 선택하면 최신 데이터 강제 조회
4. 검색 진단에 총 시간, 실제 법원요청, 캐시 사용, 대기시간, 서버 응답시간,
   브라우저 준비시간을 분리해 표시합니다.

## 예상 시간

- 최초 전북 부안군 90일 검색: 기존 약 80~100초 → 보통 약 35~50초
- 같은 조건을 15분 안에 재검색: 보통 수초 이내
- 현행 코드가 0건이어서 과거 코드와 시·도 대체검색까지 필요한 경우에는 더 오래 걸릴 수 있습니다.

법원 사이트 차단을 피하기 위해 요청 간 기본 3초와 무작위 대기시간은 유지합니다.
`수집·알림 설정`에서 대기시간을 지나치게 줄이는 것은 권장하지 않습니다.

- 직전 사건검색 화면 재사용
- 사진 실패 결과 7일 음성 캐시
