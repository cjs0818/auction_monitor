# 시·군·구 0건 및 설정파일 혼선 수정

## 확인된 문제

1. 업로드된 `/mnt/data/config.yaml`의 실제 지역은 `제주특별자치도`였습니다. 사용자가 기대한 `전북특별자치도 부안군`과 달라, GUI가 다른 작업폴더의 config.yaml을 저장하거나 사용 중인 설정파일과 업로드한 파일이 달랐을 가능성이 있습니다.
2. 법원 서버가 시·군·구 코드 검색을 0건으로 반환할 때 기존 프로그램은 결과를 복구하지 못했습니다.
3. 가격·면적·유찰·할인율을 서버에 동시에 보내면 화면검색과 내부 endpoint 결과가 달라질 수 있었습니다.

## 수정

- app.py 기준 프로젝트 폴더를 작업폴더로 고정
- GUI 사이드바에 실제 설정파일 절대경로 및 현재 실행지역 표시
- 시·군·구 검색은 지역·토지·날짜만 서버에 보내고 가격·면적·유찰·할인율은 수집 후 적용
- 시·군·구 검색이 0건이면 시·도 전체를 조회한 뒤 주소에서 선택 시·군·구만 선별
- 전북/강원 특별자치도는 현행 및 과거 지역코드 병행
- 마지막 요청/응답을 data/selenium_debug/last_search_request.json 및 last_search_response.json에 저장
- 검색 진단에 실제 실행지역, 검색방식, 지역코드, 서버 전체건수, 수집건수, 제외사유 표시
