# 온비드 공매 법정리(○○리) 주소 누락 수정

## 원인

차세대 온비드 부동산 물건목록 API는 목록 주소를 다음과 같이 나누어 제공합니다.

- `lctnSdnm`: 시·도
- `lctnSggnm`: 시·군·구
- `lctnEmdNm`: 읍·면·동
- `ltnoPnu`: 19자리 PNU

PNU에서는 지번 숫자와 산 여부를 복원할 수 있지만, 법정리의 **이름**은 별도 코드표 없이는 바로 알 수 없습니다. 온비드 물건명에는 대체로 `변산면 도청리 100-2`처럼 법정리와 지번이 함께 들어가는데, 기존 정규식은 읍·면 다음에 지번이 바로 오는 형식만 처리하여 중간의 `도청리`를 누락했습니다.

## 수정 내용

- `읍/면 + 법정리 + 지번` 형식 인식
- `시·도 + 시·군·구 + 읍/면 + 법정리 + 지번` 전체주소 인식
- 물건명에 법정리만 있고 지번이 없으면:
  - 물건명에서 법정리 추출
  - PNU에서 산 여부·본번·부번 복원
  - 두 정보를 결합해 정확한 지번주소 생성
- 상세화면 원시정보에 `복원 법정리` 추가

## 주소 결정 우선순위

1. 상세 API의 지번주소 `zadrNm`
2. 구 API의 지번주소 `LDNM_ADRS`
3. 물건명에 포함된 `읍/면 + 리 + 지번`
4. 물건명의 법정리 + PNU 복원 지번
5. 읍·면·동 + PNU 복원 지번
6. 도로명주소

## 적용 후 예시

수정 전:

```text
전북특별자치도 부안군 변산면 100-2
```

수정 후:

```text
전북특별자치도 부안군 변산면 도청리 100-2
```

산 지번 예시:

```text
전북특별자치도 부안군 변산면 격포리 산 12-3
```

패치를 적용한 뒤 공매를 다시 검색하면 기존 저장 물건도 새 주소로 갱신됩니다.
