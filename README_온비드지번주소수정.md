# 온비드 공매 지번주소 누락 수정

## 원인

차세대 온비드 부동산 물건목록 API는 목록 응답에서 주소를 다음 필드로 나누어 제공합니다.

- `lctnSdnm`: 시·도
- `lctnSggnm`: 시·군·구
- `lctnEmdNm`: 읍·면·동
- `ltnoPnu`: 토지 PNU

기존 프로그램은 앞의 세 주소명만 합쳐 사용했기 때문에 지번 숫자가 표시되지 않는 물건이 있었습니다.

## 수정 내용

- `ltnoPnu` 19자리에서 다음 정보를 자동 해석합니다.
  - 일반/산 구분
  - 본번
  - 부번
- 예시
  - 일반번지: `...1 0100 0000` → `100`
  - 산번지: `...2 0012 0003` → `산 12-3`
- 주소 선택 우선순위
  1. 상세 API의 지번주소 `zadrNm`
  2. 구 API의 지번주소 `LDNM_ADRS`
  3. 물건명에 포함된 전체 지번주소
  4. 시·도 + 시·군·구 + 읍·면·동 + PNU 복원 지번
  5. 도로명주소
- 상세화면 원시정보에 다음 항목을 표시합니다.
  - 지번 PNU
  - 복원 지번
  - 주소 확인 방식

## 적용 후 사용

패치를 적용하고 프로그램을 재시작한 뒤 공매를 다시 검색하십시오.
기존 온비드 캐시가 남아 있어도 원시 응답을 다시 정규화하므로 PNU가 포함된 캐시는 그대로 활용할 수 있습니다.

기존 DB에 저장된 과거 주소는 동일 물건을 다시 검색하면 갱신됩니다.
